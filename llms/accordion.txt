# JamUI — Accordion

> A monospace, high-contrast, fully accessible accordion component for Svelte 5 (Runes) and Tailwind CSS.

## Accordion API & Usage Guide

The JamUI Accordion is a set of Svelte 5 components that build an accessible, multi-item collapsible panel. It uses Svelte 5 Runes (`$state`, `$derived`, `$props`, `$bindable`, `untrack`) and context-based state management for seamless parent-child communication. Supports single and multiple expansion modes, five visual variants, three sizes, and custom icon rendering via snippets.

### Component Structure

- `Accordion`: Root container that holds all state and provides context to children.
- `AccordionItem`: Wrapper for a single accordion item with its own open/closed state.
- `AccordionTrigger`: The clickable header/button that toggles the item open or closed.
- `AccordionContent`: The expandable panel containing the item's content.

---

## Components API

### Accordion.svelte

Root wrapper component that manages accordion state and distributes it to children via Svelte context.

#### Props
- `type`: `'single' | 'multiple'` (default: `'single'`). Determines if one or multiple items can be open at the same time.
- `value`: `string | string[]` (bindable). The currently active item value(s). Controlled mode.
- `defaultValue`: `string | string[]` (default: `[]`). Initial active item value(s). Used in uncontrolled mode.
- `onValueChange`: `(value: string | string[]) => void`. Callback fired when the active item changes.
- `collapsible`: `boolean` (default: `false`). If `true` and `type` is `'single'`, allows closing the active item by clicking it again.
- `disabled`: `boolean` (default: `false`). If `true`, disables all accordion items.
- `variant`: `'default' | 'bordered' | 'separated' | 'filled' | 'ghost'` (default: `'default'`). Visual styling variant applied to all items.
- `size`: `'sm' | 'md' | 'lg'` (default: `'md'`). Controls padding and font-size of triggers and content.
- `icon`: `Snippet<[boolean]>` (optional). Custom Svelte 5 snippet for the trigger icon, receives the open state boolean as argument.
- `children`: `Snippet`. AccordionItem components.
- `class`: `string` (default: `''`). Additional CSS classes.

---

### AccordionItem.svelte

Wrapper for each accordion item. Must be a child of `Accordion`.

#### Props
- `value`: `string` (required). A unique identifier for the item.
- `disabled`: `boolean` (default: `false`). Disables this specific item.
- `children`: `Snippet`. AccordionTrigger and AccordionContent components.
- `class`: `string` (default: `''`). Additional CSS classes.

---

### AccordionTrigger.svelte

The toggle button for the item. Must be a child of `AccordionItem`.

#### Props
- `headingLevel`: `1 | 2 | 3 | 4 | 5 | 6` (default: `3`). Renders the trigger inside a semantic heading tag (e.g., `<h3>`).
- `children`: `Snippet`. Trigger label content.
- `class`: `string` (default: `''`). Additional CSS classes.

---

### AccordionContent.svelte

The expandable panel containing the content. Must be a child of `AccordionItem`.

#### Props
- `children`: `Snippet`. Content to render inside the panel.
- `class`: `string` (default: `''`). Additional CSS classes.

---

## Design System Tokens & Styling

JamUI uses Tailwind CSS utility classes and custom theme tokens:
- **Background**: `#1713c8` (`bg-jam-bg`)
- **Surface**: `#1410bd` (`bg-jam-surface`)
- **Input Surface**: `#0f0ca4` (`bg-jam-input`)
- **Primary Text**: `#fbfbff` (`text-jam-text`)
- **Muted Text**: `#d8d7ff` (`text-jam-muted`)
- **Mono Font**: `font-jam-mono`
- **Hard Shadow**: `shadow-jam-hard-sm`, `shadow-jam-hard-md`

#### Variants Style mapping:
- `default`: Underline border (`border-b border-jam-bg/20`)
- `bordered`: Complete border, margin bottom (`border border-jam-bg/30 mb-2`)
- `separated`: Complete border, margin bottom, surface background (`border border-jam-bg/30 mb-3 bg-jam-surface`)
- `filled`: Surface background, margin bottom (`bg-jam-surface mb-2`)
- `ghost`: Margin bottom, no borders/background (`mb-1`)

#### Size Classes (Trigger & Content):
- `sm`: `px-3 py-2 text-xs`
- `md`: `px-4 py-3 text-sm`
- `lg`: `px-5 py-4 text-base`

## Examples

### Basic Single Accordion
```svelte
<script lang="ts">
  import { Accordion, AccordionItem, AccordionTrigger, AccordionContent } from 'jamui';
</script>

<Accordion type="single" collapsible>
  <AccordionItem value="item-1">
    <AccordionTrigger>Is it accessible?</AccordionTrigger>
    <AccordionContent>
      Yes, it follows WAI-ARIA patterns.
    </AccordionContent>
  </AccordionItem>
  <AccordionItem value="item-2">
    <AccordionTrigger>Is it styled?</AccordionTrigger>
    <AccordionContent>
      Yes, with JamUI high-contrast aesthetic.
    </AccordionContent>
  </AccordionItem>
</Accordion>
```

### Multiple Accordion with custom binding and variants
```svelte
<script lang="ts">
  import { Accordion, AccordionItem, AccordionTrigger, AccordionContent } from 'jamui';

  let activeValues = $state(['item-1']);
</script>

<Accordion type="multiple" bind:value={activeValues} variant="separated" size="lg">
  <AccordionItem value="item-1">
    <AccordionTrigger>Section 1</AccordionTrigger>
    <AccordionContent>Content for section 1.</AccordionContent>
  </AccordionItem>
  <AccordionItem value="item-2">
    <AccordionTrigger>Section 2</AccordionTrigger>
    <AccordionContent>Content for section 2.</AccordionContent>
  </AccordionItem>
</Accordion>
```

### Custom Icon Accordion
```svelte
<script lang="ts">
  import { Accordion, AccordionItem, AccordionTrigger, AccordionContent } from 'jamui';
  import Plus from 'lucide-svelte/icons/plus';
  import Minus from 'lucide-svelte/icons/minus';
</script>

{#snippet customIcon(isOpen)}
  {#if isOpen}
    <Minus size={16} />
  {:else}
    <Plus size={16} />
  {/if}
{/snippet}

<Accordion type="single" icon={customIcon}>
  <AccordionItem value="item-1">
    <AccordionTrigger>Accordion with Custom Icon</AccordionTrigger>
    <AccordionContent>
      This accordion uses a custom plus/minus icon snippet.
    </AccordionContent>
  </AccordionItem>
</Accordion>
```

## Accessibility & Interactions

- **Keyboard Navigation**:
  - `ArrowDown`: Moves focus to the next enabled trigger.
  - `ArrowUp`: Moves focus to the previous enabled trigger.
  - `Home`: Moves focus to the first enabled trigger.
  - `End`: Moves focus to the last enabled trigger.
- **ARIA attributes managed**:
  - `role="region"` and `aria-labelledby` on the content container.
  - `aria-expanded` and `aria-controls` on the trigger button.
  - `inert` and `aria-hidden` attributes applied correctly on closed state to prevent tab navigation into hidden panels.
  - `data-state="open" | "closed"` on both item and content for styling hooks.
  - `data-disabled` on disabled items.
