## AlertDialog API & Usage Guide

The JamUI AlertDialog is a modal dialog designed for important user decisions that require acknowledgment or confirmation. It uses Svelte 5 Runes (`$state`, `$derived`, `$props`, `$bindable`) and context-based state management for parent-child communication. The dialog manages its own open/close animations, focus trapping, and keyboard interactions. It is composed of composable sub-components: Content, Header, Title, Description, Footer, Action, and Cancel.

### Component Structure

- `AlertDialogContent`: Root container managing open state, overlay, panel, focus trap, and animations.
- `AlertDialogHeader`: Flex container for the title and description.
- `AlertDialogTitle`: The dialog heading rendered as an `<h2>`.
- `AlertDialogDescription`: The dialog description paragraph.
- `AlertDialogFooter`: Flex container for action buttons, aligned to the right.
- `AlertDialogAction`: Primary action button that closes the dialog on click.
- `AlertDialogCancel`: Secondary cancel button that closes the dialog on click.

## Components API

### AlertDialogContent.svelte

Root container that renders the overlay and dialog panel. Manages open/close state, animations, and focus trapping.

#### Props
- `open`: `boolean` (bindable, default: `false`). Controls whether the dialog is open.
- `onopenchange`: `(open: boolean) => void`. Callback fired when the dialog open state changes.
- `children`: `Snippet`. Dialog sub-components (Header, Footer, etc.).
- `class`: `string` (default: `''`). Additional CSS classes merged with the panel styles.

---

### AlertDialogHeader.svelte

Flex container for the dialog title and description.

#### Props
- `children`: `Snippet`. Title and description sub-components.
- `class`: `string` (default: `''`). Additional CSS classes.

---

### AlertDialogTitle.svelte

The dialog heading. Renders as an `<h2>` element.

#### Props
- `children`: `Snippet`. Title text content.
- `class`: `string` (default: `''`). Additional CSS classes.

---

### AlertDialogDescription.svelte

The dialog description. Renders as a `<p>` element.

#### Props
- `children`: `Snippet`. Description text content.
- `class`: `string` (default: `''`). Additional CSS classes.

---

### AlertDialogFooter.svelte

Flex container for the dialog action buttons, aligned to the right.

#### Props
- `children`: `Snippet`. Action and cancel buttons.
- `class`: `string` (default: `''`). Additional CSS classes.

---

### AlertDialogAction.svelte

Primary action button. Closes the dialog when clicked.

#### Props
- `children`: `Snippet`. Button label content.
- `class`: `string` (default: `''`). Additional CSS classes.
- All standard `HTMLButtonAttributes` are forwarded.

---

### AlertDialogCancel.svelte

Secondary cancel button. Closes the dialog when clicked.

#### Props
- `children`: `Snippet`. Button label content.
- `class`: `string` (default: `''`). Additional CSS classes.
- All standard `HTMLButtonAttributes` are forwarded.

## Examples

### Basic Confirmation Dialog
```svelte
<script lang="ts">
  import {
    AlertDialogContent,
    AlertDialogHeader,
    AlertDialogTitle,
    AlertDialogDescription,
    AlertDialogFooter,
    AlertDialogAction,
    AlertDialogCancel
  } from 'jamui';

  let open = $state(false);
</script>

<button onclick={() => open = true} class="jam-button">
  Delete Account
</button>

<AlertDialogContent bind:open>
  <AlertDialogHeader>
    <AlertDialogTitle>Are you sure?</AlertDialogTitle>
    <AlertDialogDescription>
      This action cannot be undone. Your account and all data will be permanently deleted.
    </AlertDialogDescription>
  </AlertDialogHeader>
  <AlertDialogFooter>
    <AlertDialogCancel>Cancel</AlertDialogCancel>
    <AlertDialogAction>Delete</AlertDialogAction>
  </AlertDialogFooter>
</AlertDialogContent>
```

### Destructive Action with Custom Styling
```svelte
<script lang="ts">
  import {
    AlertDialogContent,
    AlertDialogHeader,
    AlertDialogTitle,
    AlertDialogDescription,
    AlertDialogFooter,
    AlertDialogAction,
    AlertDialogCancel
  } from 'jamui';

  let open = $state(false);

  function handleDelete() {
    console.log('Item deleted');
    open = false;
  }
</script>

<button onclick={() => open = true}>Remove Item</button>

<AlertDialogContent bind:open class="max-w-sm">
  <AlertDialogHeader>
    <AlertDialogTitle>Remove Item</AlertDialogTitle>
    <AlertDialogDescription>
      This will remove the item from your workspace. Team members will lose access.
    </AlertDialogDescription>
  </AlertDialogHeader>
  <AlertDialogFooter>
    <AlertDialogCancel>Go Back</AlertDialogCancel>
    <AlertDialogAction onclick={handleDelete}>Confirm Remove</AlertDialogAction>
  </AlertDialogFooter>
</AlertDialogContent>
```

### Simple Acknowledge Dialog


## Accessibility & Interactions

- **Keyboard Navigation**:
  - `Escape`: Closes the dialog (stops propagation).
  - `Tab`: Traps focus within the dialog panel. Wraps from last to first focusable element, and from first to last with Shift+Tab.
- **ARIA attributes managed**:
  - `role="alertdialog"` on the dialog panel.
  - `aria-modal="true"` on the dialog panel.
  - `tabindex="-1"` on the dialog panel for programmatic focus.
  - `aria-hidden="true"` on the overlay.
  - Focus is automatically moved to the first focusable element when opened.
  - Previous active element is restored when the dialog closes.
- **Animations**:
  - Overlay fades in/out with `transition-opacity duration-200`.
  - Panel scales and fades in/out with `transition-all duration-200`.
